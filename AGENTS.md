# Power Platform Terraform Module Agent Guide

This repository is a **Terraform module** for Power Platform, following AVM-aligned conventions. This is a template repository — `main.tf` contains placeholder comments, not real resources.

## Tech Stack

- **Terraform**: >= 1.9, < 2.0
- **Provider**: `microsoft/power-platform` ~> 4.0
- **Security scanner**: Trivy (configured via `.trivy.yaml`)
- **Documentation**: terraform-docs (configured via `.terraform-docs.yml`)
- **Build tool**: GNU Make (see `Makefile`)

## Module Naming Conventions

- `res-*` — Resource modules (single resource type)
- `ptn-*` — Pattern modules (multi-resource patterns)
- `utl-*` — Utility modules (helpers and data lookups)

## Repository Structure

| Path                        | Purpose                                                              |
| --------------------------- | -------------------------------------------------------------------- |
| `main.tf`                   | Primary resource definitions                                         |
| `variables.tf`              | Input variable declarations (every var needs `description` + `type`) |
| `outputs.tf`                | Output value declarations (every output needs `description`)         |
| `locals.tf`                 | Local value computations                                             |
| `terraform.tf`              | Provider and Terraform version constraints                           |
| `examples/basic/`           | Minimal usage example                                                |
| `examples/complete/`        | Full-featured usage example                                          |
| `tests/unit/`               | Mock-provider unit tests (`.tftest.hcl`)                             |
| `tests/integration/`        | Real-provider integration tests (`.tftest.hcl`)                      |
| `_header.md` / `_footer.md` | terraform-docs partials for README generation                        |

## Essential Commands

```bash
# Validate
terraform init -backend=false && terraform validate

# Run everything before committing (fmt, validate, docs, lint, security-scan, test-unit)
make check-all
```

## Variable Rules

- Every variable MUST have `description` and `type`
- Use `validation` blocks for constrained inputs
- Mark sensitive values with `sensitive = true`
- Use `nullable = false` when null is not valid
- Optional inputs SHOULD have a `default` value representing the most secure, governance-aligned configuration
- Sensitive inputs MUST NOT have a `default` value
- MUST NOT create variables named `enabled` or `module_depends_on` (AVM TFNFR14)

## Output Rules

- Every output MUST have a `description`
- Mark sensitive outputs with `sensitive = true`
- Output resource ID and name at minimum

## Boundaries

### Always
- Follow AVM TFNFR8 argument ordering in resource/data blocks
- Use `lower_snake_case` for all identifiers
- Add `validation` blocks for variables with constrained inputs
- Mark sensitive variables and outputs with `sensitive = true`
- Use pessimistic version constraints (`~>`) for providers
- Run `make check-all` before finalizing changes
- Use OIDC authentication patterns (no client secrets)
- Use `mock_provider "powerplatform"` for unit tests
- Pin GitHub Actions to full commit SHA with version comment

### Ask First
- Adding new provider dependencies beyond `microsoft/power-platform`
- Creating new submodules in `modules/`
- Modifying `terraform.tf` version constraints
- Suppressing Trivy security findings in `.trivy.yaml`
- Changing the module interface (adding/removing/renaming variables or outputs)

### Never
- Hardcode credentials, secrets, API keys, or tenant IDs in any `.tf` file
- Edit `README.md` directly (it is auto-generated by terraform-docs)
- Create variables named `enabled` or `module_depends_on` (AVM TFNFR14)
- Use `any` type for variables unless absolutely necessary
- Declare `provider` blocks inside modules (except `configuration_aliases`)
- Use git references for module sources — use Terraform registry with pinned versions
- Remove or weaken existing `validation` blocks without justification
- Commit `.terraform/`, `*.tfstate`, or `*.tfplan` files

| --- | --- | --- |
| **Module Lifecycle** | [`module-lifecycle.md`](.github/agents/module-lifecycle.md) | End-to-end module development — implement, test, document, validate, and commit changes from a requirement |
